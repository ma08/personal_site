# Repository Guidelines

## Project Structure & Module Organization
- `content/` Markdown pages and posts. Prefer `content/blog/YYYY-MM-DD-title.md`; sections like `about/index.md`, `resume/index.md`.
- `layouts/` Hugo templates and overrides. Do not edit `themes/PaperMod` directly; override via `layouts/`.
- `assets/` Pipelines for CSS/JS; custom styles in `assets/css/extended/custom.css`.
- `static/` Published as-is (e.g., `static/images/`, `static/downloads/`).
- `public/` Build output (generated by `hugo`). Never edit by hand.
- `themes/PaperMod` Theme submodule. Keep changes out of this directory.
- `scripts/` Dev/deploy helpers. Config in `hugo.yaml`. Archetypes in `archetypes/`.

## Build, Test, and Development Commands
- Local preview: `./scripts/run-local.sh` (Hugo server with drafts and ngrok baseURL).
- Ngrok tunnel: `./scripts/run-ngrok.sh` (uses `ngrok.yml`).
- Production build: `hugo` (outputs to `public/`).
- Vercel build: `./scripts/vercel-build.sh` (updates submodules, runs `hugo`).
- AWS deploy: `./scripts/deploy-to-s3.sh` (requires `.env` from `.env.template`).
- Theme sync: `git submodule update --init --recursive`.

## Coding Style & Naming Conventions
- Markdown with YAML front matter (`---`). Use 2-space indentation in YAML.
- Filenames: kebab-case, lowercase (e.g., `content/blog/2025-04-13-new-site-who-this.md`).
- Front matter: include `title`, `date`, `draft`, and optional `tags`, `summary`.
- Assets: place images in `static/images/`; link as `![alt](/images/file.jpg)`.
- CSS/JS: extend via `assets/` and Hugo Pipes; avoid editing theme files.

## Testing Guidelines
- Preview with `hugo server -D`; verify layout, links, images.
- Before publishing: set `draft: false`, run `hugo`, and spot-check `public/` output.
- No unit tests required; treat a clean build and visual review as acceptance.

## Commit & Pull Request Guidelines
- Commits: concise, present tense. Prefer Conventional Commits (`feat:`, `fix:`, `docs:`, `style:`, `chore:`).
- PRs: include a clear description, linked issues/URLs, and screenshots/GIFs for visual changes; note config/env updates.

## Security & Configuration Tips
- Never commit secrets. Copy `.env.template` to `.env` and fill locally.
- Do not edit `public/` or `themes/PaperMod`; make changes via `content/`, `layouts/`, or `assets/`.
- Review `hugo.yaml` for site-wide settings before proposing structural changes.
